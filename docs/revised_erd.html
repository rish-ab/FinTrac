PORTFOLIO NEW
  • Separates user goals — retirement vs speculative vs income
  • TRANSACTION_LEDGER and POSITION_JOURNAL now FK into this
  • base_currency field fixes multi-currency P&L aggregation
  • objective enum: GROWTH / INCOME / PRESERVATION / SPECULATIVE
MACRO_INDICATOR_MASTER NEW
  • Fixes CRITICAL: PARQUET_MACRO_SERIES was a table of unidentified numbers
  • series_code maps to FRED / BLS / ECB identifiers (e.g. FEDFUNDS, CPIAUCSL)
  • unit and release_frequency allow correct normalisation in AI prompts
  • country_code enables geo-filtered macro queries
RECOMMENDATION_FEEDBACK NEW
  • Fixes CRITICAL: AI had no feedback loop — could never improve
  • user_acted boolean tracks if recommendation was followed
  • outcome_pct_7d and outcome_pct_30d measure prediction accuracy
  • user_rating (1–5) captures subjective satisfaction
COMPARISON_SESSION NEW
  • Persists the core user workflow: "compare AAPL vs GOOGL vs 2yr Bond"
  • budget and horizon_months carry the user's constraint context
  • assets_compared (JSON array) stores all evaluated options
  • winner_asset_id records AI's final recommendation
ALERT_QUEUE NEW
  • WATCHLIST had trigger thresholds with nowhere to send them
  • channel enum: EMAIL / PUSH / WEBHOOK / SMS
  • delivered_at enables retry logic for failed deliveries
  • Decouples detection from delivery for async processing
DOCUMENT_REGISTRY NEW
  • Required for RAG: AI must trace sentiment back to source filing
  • doc_type enum: SEC_8K / SEC_10K / EARNINGS_CALL / NEWS / ANALYST_REPORT
  • raw_text_path links to the extracted text file for vector embedding
  • embedding_status tracks RAG pipeline progress: PENDING / DONE / FAILED
PARQUET_MANIFEST NEW
  • Replaces unenforced cross-store FKs with a verifiable audit trail
  • schema_hash detects silent schema drift in Parquet files
  • row_count enables reconciliation jobs to catch missing data
  • validation_status tracks nightly reconciliation results
ASSET_CLASS_LOOKUP NEW
  • Normalises free-text asset_class string in ASSET_MASTER
  • comparable_with JSON array guides valid AI comparisons
  • Prevents AI from comparing CRYPTO with BOND_GOVT without caveats
ASSET_SUBTYPE_BOND / FOREX NEW
  • Bond-specific: maturity_date, coupon_rate, issuer_type, credit_rating
  • Forex-specific: base_currency, quote_currency
  • Keeps ASSET_MASTER generic while storing type-specific data correctly
ORDER_GROUP NEW
  • Replaces string transaction_group_id — now a proper FK entity
  • strategy_label tags multi-leg trades (e.g. "Pairs Trade AAPL/MSFT")
  • Referential integrity now enforceable at DB level
ASSET_MASTER — Updated
  • Fixes CRITICAL: added exchange and denomination_currency
  • sector and industry fields for AI peer comparison ("suggest similar tech large cap")
  • country_code for geopolitical risk filtering
  • asset_class_id now FKs to ASSET_CLASS_LOOKUP instead of free string
PARQUET_TICK_DATA — Updated
  • Fixes CRITICAL: replaced single price with full OHLCV schema
  • adjusted_close handles stock splits and dividend adjustments
  • bid and ask fields for spread and execution cost modelling
  • Enables RSI, MACD, Bollinger Bands, and all standard TA indicators
PARQUET_NLP_SENTIMENT — Updated
  • source_document_id replaced by document_id FK to DOCUMENT_REGISTRY
  • source_type enum: NEWS / SEC_FILING / EARNINGS_CALL / SOCIAL_MEDIA
  • positive_score and negative_score stored separately alongside compound
  • Allows source-weighted sentiment (8-K > Reddit post)
AI_RECOMMENDATION_LOG — Updated
  • Fixes CRITICAL: model_version added for reproducibility
  • Fixes CRITICAL: context_snapshot_id links to data used at inference time
  • expires_at prevents stale recommendations surfacing in UI
  • comparison_session_id FK traces which session triggered this recommendation
RISK_SETTINGS — Updated
  • risk_tolerance_level: CONSERVATIVE / MODERATE / AGGRESSIVE / SPECULATIVE
  • investment_horizon_months replaces implicit assumption
  • liquidity_preference and sector_exclusions feed AI prompt directly
  • valid_from / valid_to enables SCD Type 2 history of risk profile changes
POSITION_JOURNAL — Updated
  • is_current boolean + partial unique index prevents duplicate open rows
  • realized_pnl and unrealized_pnl critical for AI portfolio health assessment
  • position_currency fixes multi-currency P&L aggregation bug
  • portfolio_id FK connects positions to user goals
